■起動するために必要なこと
・初回起動の際はdefault_scoreフォルダをscoreにリネームします。
・ローカル接続(クライアントとサーバーが同一PC)であればそのまま起動するだけでOK。
・それ以外の場合(外部接続の受け入れ)は、一度起動して終了するとiniファイルが生成されるので、
　この中のGlobalIPに自身のグローバルIPアドレスを設定し起動することで使用できます。
・クライアントからの接続にはWinYG(※)を使用します。
　接続の際は「接続先を変更」にチェックし、その下に接続先アドレスを入力し起動します。
　接続先アドレスには、自分で起動したサーバーに繋ぐ時は「127.0.0.1」を、
　他の人が立てたサーバに繋ぐ時は教えてもらったグローバルIPアドレスを入力します。
※WinYGの入手先: http://twc.xrea.jp/junk/yogurting/

■接続を受け入れるために必要なこと
・使用ポートはTCPの10000〜10004です。
　ルーターなどを使用している場合はこれらを開放しておく必要があります。
　開放の方法については各自ルーターの説明書などを読んでください。

■注意点

■デバッグコマンド
・コマンドはチャット発言欄で、コマンド名に/を付けて発言する。
　(以前はコマンド接頭辞は@だったが、build090503から変わったので注意)
　パラメータは基本的にスペース区切りで渡す。[]内は省略可能。
　パラメータにスペースを含みたい場合は""でくくる(ex. /a "test message")。

■デバッグコマンドの注意事項
デバッグ用のコマンドという性質上、複数クライアントが接続した環境で使用した場合、
一方にしか効果が出ない、不整合が出る、最悪サーバの進行が止まる場合があるので注意。
特にspawnなどはパラメータの範囲チェック等全くしていないのでクライアントが操作不能になる場合も。

■キャプションに表示されるspf値について
負荷率のようなもの。0〜30までは問題なし。
それを超えるとラグなどの処理落ちが発生する可能性がある。
(敵を全マップに配置した状態で全マップにログインしたキャラがいた場合、簡単に超えます)

■GM権限の取得について
現在GM権限を取得する方法は直接保存されたデータを書き換えます。
まずサーバをCtrl+Cなどで終了し、データを保存します。
次にSaveフォルダのAccount.xmlを開き、該当キャラのAuthTypeというタグを探します。
値が"none"になっているので、これを"gm"に書き換えます。これでGM権限を付与できます。
admintoolを使用することでもGM権限を操作できます。

■iniの設定項目
[LocalIP]
LAN内から接続する際はここにサーバPCのアドレス(192.168.xxx.xxx)を指定します。

[GlobalIP]
外部からの接続をする際はここにサーバPCのグローバルアドレスを指定します。ホスト名も可。

[LoginPort]
ログインサーバのポート番号を指定します。デフォルトでは10000です。
この値を変えた場合、WinYGの接続先指定は「IP:Port」とします(ex. 127.0.0.1:10001 など)。
通常変更する必要はありません。

[SchoolPort]
スクールサーバのポート番号を指定します。デフォルトでは10002です。
通常変更する必要はありません。

[AttractionPort]
アトラクションサーバのポート番号を指定します。デフォルトでは10003です。
通常変更する必要はありません。

[CommPort]
コミュサーバのポート番号を指定します。デフォルトでは10004です。
通常変更する必要はありません。

[AdminPort]
管理サーバのポート番号を指定します。デフォルトでは10010です。
通常変更する必要はありません。

[LocalTest]
0=外部接続を受け付けます 1=ループバックアドレス(127.0.0.1)、プライベートアドレスのみ受け付けます。

[WorldName]
ワールド名を指定します。文字数(≠バイト数)で10文字まで可。

[NoSave]
キャラの状態や設定の保存を制御します。
0=保存する(デフォルト) 1=保存しない

[SaveInterval]
データの定期保存間隔をミリ秒で指定します。最小値は5分(300000)です。
NoSaveが有効な場合保存されません。デフォルトでは10分(600000)です。

[MakeAccount]
ログイン時にIDの先頭に"make_"を付けるとアカウント作成できる機能を制御します。
0=無効 1=有効(デフォルト)

[InOutNotify]
キャラクターのログイン・ログアウト時に通知します。
0=無効 1=有効(デフォルト)

[AuthPassword]
外部からの認証で使うパスワードを設定します。
変更せずに使用して不正アクセスを受けないようにするため、
初期値はランダムな文字列になっています。
パスワード長は1〜32767文字です。

[OutputError]
エラーメッセージのファイル出力を制御します。
0=出力しない 1=出力する(デフォルト)

[KujiTepmlate]
kujiコマンドで表示する文章を指定します。
[[NAME]] という記述はキャラ名に、[[RESULT]] という記述はkuji.txtの内容にそれぞれ置き換えられます。

[DefaultShout]
shoutコマンドの初期状態を指定します。
有効にした場合、ログイン時からshoutコマンドが実行された状態になります。
0=無効(デフォルト) 1=有効

[GeneratorRespawnTime]
ジェネレータから敵が出現する間隔を秒で指定します。デフォルトでは5秒です。

[FieldSpawnLimit]
フィールドに存在できる敵の最大数を指定します。デフォルトは200です。
この値を変えると負荷が極端に大きくなることがあります。
